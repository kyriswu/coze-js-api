# File-transfer 上传完整链接设计

## 目标

上传完成后，`file-transfer` 接口返回可直接访问的完整文件链接。

## 决策

- 链接按当前请求的协议和 Host 动态生成，不配置或固定外部域名。
- 保持既有 `data.url` 字段与响应结构兼容。
- 普通上传和分片合并使用同一链接生成逻辑。
- 文件名继续 URL 编码；临时文件使用 `/downloads/`，永久文件使用 `/downloads/persistent/`。

## 方案取舍

1. 动态请求 Host（采用）：支持不同访问入口，符合当前部署方式。
2. 固定配置域名：单域名部署更稳定，但不符合需求且增加配置。
3. 相对路径：调用方还需自行拼接，不满足完整链接要求。

## 验证

- 静态确认普通上传和分片合并响应均调用统一的完整链接生成逻辑，链接包含协议、Host 和公开文件路径。
- 执行 `node --check index.js`；不运行服务或真实上传请求。
